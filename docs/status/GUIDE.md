# ステータスガイド

## 目的

設計書から実装ステータスを分離し、プロジェクトの進捗を一元管理する。

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `implementation.md` | モジュール別の実装状態 |
| `roadmap.md` | フェーズ別のロードマップ |

## ステータス値

| 値 | 意味 |
|----|------|
| `done` | 実装完了・動作確認済み |
| `partial` | 一部実装済み |
| `not-started` | 未着手 |

## 更新タイミング

- 新しいモジュール/機能の実装が完了したとき
- ロードマップのフェーズが完了したとき
- レビューで実装状態の変更が確認されたとき
