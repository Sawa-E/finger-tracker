# レビューガイド

## 目的

finger-tracker のコード品質を維持するためのレビュー基準とプロセスを定義する。

## ディレクトリ構成

```
review/
├── GUIDE.md           # 本ガイド
├── FORMAT.md          # 共通出力形式
├── TEMPLATE.md        # モジュール別チェックリストテンプレート
└── modules/           # モジュール別チェックリスト
    ├── capture.md
    ├── training.md
    ├── detection.md
    └── config.md
```

## 役割分担

| ファイル | 役割 |
|---------|------|
| GUIDE.md | レビュープロセス全体の定義 |
| FORMAT.md | レビュー結果の出力形式 |
| TEMPLATE.md | モジュール別チェックリストの雛形 |
| modules/*.md | モジュール固有の評価項目 |

## レビュープロセス

1. `git diff` で変更を検出
2. 対象モジュールを特定（execute-review のパスマッピング参照）
3. モジュール別チェックリストに基づきレビュー
4. FORMAT.md の形式で結果を出力

## 分類基準

| レベル | 基準 | 対応 |
|--------|------|------|
| **Critical** | マージ前に必ず修正が必要 | 必須修正 |
| **Medium** | 修正推奨。品質・保守性が向上 | 推奨修正 |
| **Low** | 将来対応の提案 | 記録・先送り可 |

## モジュール追加手順

1. `TEMPLATE.md` をコピーして `modules/{module-name}.md` を作成
2. モジュール固有の評価項目を追加
3. execute-review のパスマッピングにエントリを追加

## 記述ルール

- 評価項目は番号付きリストで記載
- 各項目に「なぜ重要か」の根拠を添える
- Python固有のベストプラクティスを含める
