# レビュー出力形式

## 形式

```markdown
# {モジュール名} レビュー結果

## Critical

- **[C-{番号}]** {指摘内容}
  - 該当箇所: `{ファイルパス}:{行番号}`
  - 理由: {なぜCriticalか}
  - 修正案: {具体的な修正方法}

## Medium

- **[M-{番号}]** {指摘内容}
  - 該当箇所: `{ファイルパス}:{行番号}`
  - 理由: {改善される点}
  - 修正案: {提案}

## Low

- **[L-{番号}]** {指摘内容}
  - 該当箇所: `{ファイルパス}:{行番号}`
  - 提案: {将来の改善案}

## 確認済み

- {問題のなかった評価項目を箇条書き}
```

## ルール

- 各レベルに該当なしの場合は「なし」と記載する
- 番号はモジュール内で通し番号にする
- 修正案は具体的なコード例を含める（可能な場合）
